Agent Smith API Documentation

Everything you need to integrate our AI-powered email personalization service with your application.

Getting Started

Agent Smith provides a REST API that allows you to programmatically submit new signups and retrieve personalized email drafts. All API requests require authentication using your API key, which you can obtain from your account dashboard.

The base URL for all API requests is: https://api.agentsmith.ai

# Example API request using curl
curl -X POST https://api.agentsmith.ai/api/process-signup \
  -H "Content-Type: application/json" \
  -H "X-API-Key: your-api-key-here" \
  -d '{"email": "john@acmecorp.com", "name": "John Smith"}'

API Endpoints

POST /api/process-signup

Processes a new signup and generates a personalized email draft based on domain research.

Requires Authentication

Request Headers

Content-Type: application/json
X-API-Key: your-api-key-here

Request Body Parameters

Parameter Type Required Description
email string Required User's email address (business email domains only)
name string Required User's full name

Response

{
  "success": true,
  "message": "Signup processing started",
  "jobId": "job_a1b2c3d4e5f6",
  "status": "pending"
}
GET /api/job-status/:jobId

Request Headers

X-API-Key: your-api-key-here

Rate Limits & Authentication

API Rate Limits

To ensure fair usage and availability of the service, all API endpoints are subject to rate limiting:

  • Standard tier: 1000 requests per day
  • Pro tier: 5000 requests per day
  • Enterprise tier: Custom limits based on your agreement

Rate limit headers are included in all API responses:

X-RateLimit-Limit: 1000
X-RateLimit-Remaining: 999
X-RateLimit-Reset: 1712545200 // Unix timestamp

Authentication

All API requests require authentication using an API key provided in the X-API-Key header:

X-API-Key: your-api-key-here

Never share your API key publicly or include it in client-side code. Your API key can be rotated from your dashboard if it is compromised.

Error Responses

In case of an error, the API will return an appropriate HTTP status code and a JSON response with error details:

{
  "success": false,
  "error": "invalid_request",
  "message": "The email parameter is required",
  "status": 400
}

Common status codes:

  • 400 - Bad Request (invalid parameters)
  • 401 - Unauthorized (invalid or missing API key)
  • 404 - Not Found (resource or endpoint not found)
  • 429 - Too Many Requests (rate limit exceeded)
  • 500 - Internal Server Error

Webhooks

Email Generation Webhook

You can configure a webhook URL in your dashboard to receive notifications when email generation is complete. This is especially useful for long-running processes where you don't want to poll the API.

Webhook Payload

{
  "event": "email.generated",
  "jobId": "job_a1b2c3d4e5f6",
  "status": "completed",
  "timestamp": "2025-04-07T14:17:45Z",
  "data": {
    "email": "john@acmecorp.com",
    "name": "John Smith",
    "domain": "acmecorp.com"
  }
}

To acknowledge receipt of the webhook, your server should respond with a 200 OK status code.

Failed webhook deliveries will be retried up to 3 times with exponential backoff.

SDK & Client Libraries

Official Client Libraries

We provide official client libraries for popular programming languages to make integration easier:

  • JavaScript/Node.js: agent-smith-node
  • Python: agent-smith-python
  • PHP: agent-smith-php
  • Ruby: agent-smith-ruby

Node.js Example

const AgentSmith = require('agent-smith-node');

const client = new AgentSmith.Client({
  apiKey: 'your-api-key-here'
});

async function processSignup() {
  try {
    const result = await client.processSignup({
      email: 'john@acmecorp.com',
      name: 'John Smith'
    });

    console.log('Job started:', result.jobId);

    // Poll for completion
    const jobResult = await client.waitForCompletion(result.jobId);
    console.log('Generated email:', jobResult.email);
  } catch (error) {
    console.error('Error:', error.message);
  }
}
